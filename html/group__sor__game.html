Detailed Description

Warning
Note: the map is drawn with extended ASCII chars, to view it properly
change translation in PuTTY:
Change Settings -> Window -> Translation
Then set the Remote character set to CP866
This game runs best at 16MHz or higher. See hal_clock

This game is a take on the classic arcade game snake. You will begin as the head of the snake '#', and will need to maneuver through the map using the 'w', 'a', 's', and 'd' keys to collect food 'X'. As you collect more food you will become an increasingly longer snake. Make sure to avoid running into the walls or your tail, as this will cause a Game Over.

When you eat food you will hear the victorious chomping of your snake indicated by the bell character.

Instructions to make this game run:

Hardware: To run this game you will need a microprocessor and an FTDI. FTDI provides a usb to serial converter with a higher performance than the onboard one. This is necessary for the game to run smoothly. This game was initially designed to be run on an MSP430F5529 processor.

Software (setting up main.c):

  1. You will need to set up a main file to run this game.
  2. Stop WDT
  3. You will need functions to configure your clock.
  4. Disable interrupts, then call Task_Init() and UART_Init()
  5. clear terminal screen
  6. call SorGame_Init
  7. Turn on Echo (must be called after Subsystem_Init()
  8. call SystemTick() in a while loop

project_settings.h will be necessary to configure game (will be specific to processor you pick) Example of this can be seen in project_settings.h

You need a terminal program to see the game. Recommended: Putty

Once game is running and showing up in console, the following can be used:

  • Command to play game: $game sor play
  • Command to see help menu: $game sor help

To see game controls in game look at the help menu.

Game controls:

  • 'w' make snake turn up
  • 'a' make snake turn left
  • 's' make snake turn down
  • 'd' make snake turn right
