Detailed Description

Operation_Maze Trap designed to simulate disabling an alarm system. Thief will have to bring a ball through a maze in a box without setting off the vibration sensor on the box.

The operation-maze is a puzzle obstacle that simulates disabling an alarm system. The trap is a wooden box with a maze cut into the top. On the base of the box lies an identical maze that is a track for a ball to roll on. The objective of the trap is to move the ball on the bottom track through the maze via the cut path on the top of the box. This top is connected to the rest of the box by springs on all four corners. When a thief attempts to move the ball through the maze with their thief tools (wooden dowel) the thief should be careful not to bump into the top layer of the box. If the thief shakes the top of the box, a vibration sensor will be set off, sending a trip message to the central node, and adding a time penalty to the thief’s overall score. This will happen each time a thief sets off the vibration sensor. When the thief is able to successfully navigate the ball to the end of the maze, the metal ball acts as a conductor to complete a circuit. This action will cause an access message to be sent to the Brainframe, while simultaneously disabling the trap.

Additionally, the operation maze can be controlled manually through the Brainframe. The Brainframe can send messages to manually arm and disarm the trap, as well as set the difficulty. The difficulty is sent as a number from 0 to 255, which will correspond to the number of seconds added as a penalty to the thief when the trap is tripped.
